# Задачи по теме: Сессии в PHP (20 вариантов)

## Вариант 1
Создайте простую форму «логина»: поля username и password. При отправке (проверка POST) сравните с тестовыми значениями (например, admin/123). При успехе сохраните $_SESSION['user_id'] и $_SESSION['username'], затем header('Location: dashboard.php') и exit. Страница dashboard выводит «Привет, {username}». Реализуйте обе страницы (или один файл с переключением).

## Вариант 2
Реализуйте функцию logout(): session_unset(), session_destroy(), header('Location: login.php'), exit. Вызовите при нажатии кнопки «Выйти» (форма с method="post", action="?logout=1"). В начале страницы проверяйте $_GET['logout'] и вызывайте logout(). Выведите сообщение «Вы вышли» перед редиректом (или сразу редирект).

## Вариант 3
Настройте cookie сессии: перед session_start() вызовите session_set_cookie_params(['lifetime' => 3600, 'path' => '/', 'secure' => true, 'httponly' => true, 'samesite' => 'Strict']). Запустите сессию и выведите «Параметры cookie установлены». Примечание: secure=true работает только по HTTPS.

## Вариант 4
Добавьте проверку «последней активности»: при каждом запросе сохраняйте $_SESSION['last_activity'] = time(). Если (time() - $_SESSION['last_activity']) > 1800 (30 мин), вызовите session_unset(), session_destroy() и редирект на login. Выведите «Сессия истекла» или продолжайте работу. Симулируйте установку last_activity в прошлое для проверки.

## Вариант 5
После успешного «логина» вызовите session_regenerate_id(true). Выведите «ID сессии обновлён». Объясните в комментарии защиту от фиксации сессии. Продемонстрируйте вызов после установки $_SESSION['user_id'].

## Вариант 6
Сохранение данных формы в сессию: при POST сохраните введённые поля в $_SESSION['form_data'] = $_POST. На следующей странице (или при повторном открытии) выведите «Ранее введено: » и данные из сессии. Очистите после вывода (unset($_SESSION['form_data'])).

## Вариант 7
Проверка авторизации: функция isLoggedIn(): bool { return isset($_SESSION['user_id']) && $_SESSION['user_id'] > 0; }. В начале защищённой страницы если (!isLoggedIn()) { header('Location: login.php'); exit; }. Выведите контент или редирект. Симулируйте отсутствие user_id.

## Вариант 8
Счётчик посещений: если не isset($_SESSION['visits']), установите 0. Увеличьте на 1. Выведите «Вы посетили страницу {visits} раз». Обновите страницу несколько раз и покажите рост счётчика.

## Вариант 9
Корзина в сессии: $_SESSION['cart'] = []. Добавьте товар: $_SESSION['cart'][] = ['id' => 1, 'name' => 'Товар', 'qty' => 1]. Реализуйте добавление через форму (product_id, qty). Выведите список товаров в корзине. Очистка корзины — unset($_SESSION['cart']); $_SESSION['cart'] = [].

## Вариант 10
Вызовите session_write_close() после того как прочитали/записали данные сессии. Выведите «Сессия закрыта для записи». Объясните в комментарии: освобождение блокировки для других запросов того же пользователя.

## Вариант 11
Хранение flash-сообщения: после действия (например, «Данные сохранены») установите $_SESSION['flash'] = 'Сохранено успешно'. На странице вывода проверьте isset($_SESSION['flash']), выведите сообщение, затем unset($_SESSION['flash']). Продемонстрируйте один цикл: установка и вывод.

## Вариант 12
Роль пользователя: при логине сохраните $_SESSION['role'] = 'admin' или 'user'. Функция hasRole(string $role): bool { return ($_SESSION['role'] ?? '') === $role; }. На странице выведите контент в зависимости от роли (админ видит ссылку «Панель», пользователь — нет). Симулируйте обе роли.

## Вариант 13
Защита от повторной отправки формы: сгенерируйте токен $_SESSION['form_token'] = bin2hex(random_bytes(16)). В форме выведите <input type="hidden" name="token" value="...">. При обработке сравните $_POST['token'] с $_SESSION['form_token']. После проверки удалите токен. Выведите «Токен верный» или «Ошибка».

## Вариант 14
Ограничение времени сессии: при создании сессии сохраните $_SESSION['created'] = time(). При каждом запросе проверяйте: если (time() - $_SESSION['created']) > 7200 (2 часа), уничтожьте сессию и редирект на login. Выведите «Сессия действительна» или выполните редирект. Симулируйте старую сессию.

## Вариант 15
Сохранение языка: форма выбора языка (ru/en). При выборе сохраните $_SESSION['lang'] = $_GET['lang']. Выведите текущий язык: «Текущий язык: {lang}». По умолчанию 'ru'.

## Вариант 16
Многоуровневое сообщение: $_SESSION['flash'] = ['type' => 'success', 'message' => 'Готово']. На странице выведите <div class="alert alert-{type}">{message}</div>, затем unset($_SESSION['flash']). Продемонстрируйте вывод.

## Вариант 17
Проверка session_status(): если session_status() === PHP_SESSION_NONE, вызовите session_start(). Выведите статус (PHP_SESSION_ACTIVE и т.д.) в комментарии. Не выводите до session_start() (никакого echo перед ним).

## Вариант 18
История просмотров: $_SESSION['history'] = []. При посещении страницы добавьте текущий URL (или id страницы) в массив, ограничьте последними 10. Выведите «Вы недавно смотрели: » и список. Симулируйте несколько «посещений» через массив.

## Вариант 19
Удаление одного ключа: unset($_SESSION['cart'][$productId]). Симулируйте корзину с двумя товарами. Удалите один по id. Выведите оставшийся массив. Продемонстрируйте unset для элемента массива в сессии.

## Вариант 20
Безопасные заголовки перед редиректом: после logout перед header('Location: ...') установите session_regenerate_id(true) и очистите сессию. Добавьте комментарий: «Редирект после уничтожения сессии предотвращает использование старого ID». Реализуйте полный сценарий выхода с редиректом на login.php.
