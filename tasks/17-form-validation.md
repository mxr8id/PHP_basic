# Задачи по теме: Проверка данных формы с помощью PHP (20 вариантов)

## Вариант 1
Валидируйте форму: поле name (обязательное, 2–50 символов), email (обязательный, формат email), age (обязательный, 18–100). Соберите ошибки в массив $errors['field']. Выведите массив ошибок или «Валидация пройдена». Симулируйте данные: один раз с ошибками, один раз корректные.

## Вариант 2
Продемонстрируйте разницу filter_var с FILTER_SANITIZE_EMAIL и FILTER_VALIDATE_EMAIL. Строка «user<>@mail.ru»: сначала sanitize, выведите результат; затем validate — false. Для «user@mail.ru» — validate true. Выведите оба результата.

## Вариант 3
Валидация пароля: минимум 8 символов, наличие буквы и цифры. Напишите проверки (strlen, preg_match для буквы, preg_match для цифры). Добавьте ошибку в $errors['password'] при несоответствии. Проверьте для «short», «onlyletters», «12345678», «Pass1234».

## Вариант 4
Создайте форму с полями name, email. При ошибках валидации выведите форму снова с предзаполненными значениями (value="<?= htmlspecialchars($name) ?>") и сообщениями об ошибках под каждым полем. Один вывод с ошибкой, один без.

## Вариант 5
Валидация URL: поле site. Используйте filter_var($url, FILTER_VALIDATE_URL). При false добавьте $errors['site']. Выведите ошибку или «URL корректен». Проверьте «http://example.com» и «not-a-url».

## Вариант 6
Поле «повтор пароля»: сравните password и password_confirm. При несовпадении добавьте $errors['password_confirm']. Выведите результат валидации.

## Вариант 7
Валидация числа в диапазоне: поле quantity, целое от 1 до 99. (int)($_POST['quantity'] ?? 0), проверка $q >= 1 && $q <= 99. Ошибка при нарушении. Выведите для 0, 50, 100.

## Вариант 8
Проверка на пустоту с trim: для каждого текстового поля сначала $value = trim($_POST['field'] ?? ''). Затем empty($value). Продемонстрируйте на поле «комментарий»: пустая строка и строка из пробелов — обе дают ошибку.

## Вариант 9
Валидация телефона: допустимы цифры, пробелы, +, -, скобки. Минимум 10 цифр. Используйте preg_match или подсчёт цифр. Добавьте ошибку при несоответствии. Проверьте «+7 (999) 123-45-67» и «abc».

## Вариант 10
Чекбокс «подписка»: значение только если отмечен. Проверьте isset($_POST['subscribe']). При отмеченном — сохраните в $data['subscribe'] = true. Выведите $data после валидации (без ошибок полей).

## Вариант 11
Множественный выбор (select multiple или checkboxes): допустимые значения из списка ['a', 'b', 'c']. Получите массив из $_POST. Проверьте каждый элемент in_array($value, $allowed). При недопустимом значении добавьте ошибку. Выведите ошибки или «Выбрано: ...».

## Вариант 12
Валидация даты: поле birthday в формате Y-m-d. Проверьте через strtotime или DateTime::createFromFormat. Если дата невалидна или будущая — ошибка. Выведите результат для «2020-01-15» и «invalid».

## Вариант 13
Длина строки: поле «название» от 3 до 100 символов. strlen после trim. Добавьте разные сообщения: «Слишком короткое», «Слишком длинное». Выведите для строк длиной 2, 50, 150.

## Вариант 14
Санитизация перед сохранением: после валидации сохраните в $data только пройденные поля. Для email используйте filter_var(..., FILTER_SANITIZE_EMAIL). Для имени — htmlspecialchars. Выведите $data (без пароля).

## Вариант 15
Валидация «ник»: только латинские буквы, цифры, подчёркивание. Длина 3–20. preg_match('/^[a-zA-Z0-9_]{3,20}$/', $nick). Ошибка при несоответствии. Проверьте «user_1», «U», «user name».

## Вариант 16
Поле «сумма»: число, два знака после запятой, от 0.01 до 999999.99. (float) или фильтр, проверка диапазона. Ошибка при 0 или отрицательном. Выведите для 10.5 и -5.

## Вариант 17
Объедините валидацию: имя (обязательное, 2–50), email (обязательный, формат), возраст (18–100). Если массив $errors пуст — установите $success = true и выведите «Данные сохранены». Иначе выведите список ошибок по полям.

## Вариант 18
Валидация выбора из списка: поле role, допустимые ['user', 'admin', 'moderator']. Получите $_POST['role']. Если не in_array — ошибка. Выведите для 'admin' и 'hacker'.

## Вариант 19
Проверка на целое число: поле count. filter_var($value, FILTER_VALIDATE_INT). При false — ошибка. Дополнительно диапазон 1–100. Выведите для «5», «5.5», «0».

## Вариант 20
Форма с полем «капча» (упрощённо): ожидаемое значение хранится в сессии. Сравните $_POST['captcha'] с $_SESSION['captcha']. При несовпадении добавьте $errors['captcha']. Выведите «Капча верна» или «Ошибка капчи». Симулируйте сессию и POST.
