# Задачи по теме: Взаимодействие PHP и MySQL (20 вариантов)

## Вариант 1
Подключитесь к MySQL через PDO. DSN с host, dbname, charset=utf8mb4. Опции: PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION, PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC. Выведите «Подключение успешно» или обработайте PDOException (выведите сообщение). Учётные данные возьмите из переменных или getenv.

## Вариант 2
Выполните SELECT с подготовленным запросом: «SELECT * FROM users WHERE id = ?». Передайте id = 1 в execute([1]). Получите одну строку fetch(). Выведите массив или «Запись не найдена». Таблица users должна существовать (или замените на свою).

## Вариант 3
Реализуйте INSERT с подготовленным запросом и именованными параметрами: INSERT INTO logs (message, created_at) VALUES (:msg, :created). bindValue для :msg и :created (date('Y-m-d H:i:s')). execute(). Выведите «Добавлено строк: » . $stmt->rowCount().

## Вариант 4
Сравните PDO и mysqli: создайте подключение в обоих вариантах (одна и та же БД). Выполните один и тот же запрос SELECT 1 (или SELECT * FROM table LIMIT 1). Выведите результат в обоих случаях. Закройте соединения.

## Вариант 5
Обработка ошибок: при подключении PDO оберните в try-catch. В catch выведите $e->getMessage() и не выводите пароль. Установите логирование ошибок (например, error_log). Симулируйте неверный пароль и выведите только «Ошибка подключения» без деталей в пользовательском выводе.

## Вариант 6
Подготовленный запрос с несколькими параметрами: SELECT * FROM products WHERE category_id = ? AND price < ?. execute([$categoryId, $maxPrice]). Выведите количество строк (fetchAll и count). Симулируйте параметры.

## Вариант 7
Используйте charset: в DSN добавьте charset=utf8mb4. Для mysqli после подключения вызовите set_charset('utf8mb4'). Выведите «Charset установлен». Выполните простой SELECT и выведите одну строку (если есть таблица с кириллицей — проверьте кодировку).

## Вариант 8
UPDATE с подготовленным запросом: UPDATE users SET name = ? WHERE id = ?. execute([$name, $id]). Выведите rowCount() — сколько строк обновлено. Симулируйте данные.

## Вариант 9
Проверка существования таблицы: выполните запрос SHOW TABLES LIKE 'users'. Если fetch() вернёт строку — таблица есть. Выведите «Таблица users существует» или «Не найдена».

## Вариант 10
DELETE с условием: DELETE FROM temp_data WHERE id = ?. prepare, execute([$id]). Выведите количество удалённых строк. Не выполняйте на критичных данных (используйте тестовую таблицу).

## Вариант 11
Транзакция: beginTransaction(), два INSERT (или UPDATE), commit(). При исключении — rollBack(). Выведите «Транзакция выполнена» или «Откат». Симулируйте успешный сценарий.

## Вариант 12
Получите последний вставленный ID: после INSERT вызовите $pdo->lastInsertId(). Выведите «Новый ID: {id}». Таблица с AUTO_INCREMENT.

## Вариант 13
Подготовленный запрос с IN: SELECT * FROM items WHERE id IN (?, ?, ?). Массив id [1, 2, 3]. Сформируйте плейсхолдеры по длине массива (implode(',', array_fill(0, count($ids), '?'))). execute($ids). Выведите количество найденных строк.

## Вариант 14
mysqli: подготовленный запрос. $stmt = $mysqli->prepare("SELECT name FROM users WHERE id = ?"). bind_param('i', $id). execute(). get_result(), fetch_assoc(). Выведите имя или «Не найдено».

## Вариант 15
Проверка подключения без вывода пароля: при ошибке PDO выведите «Ошибка подключения к БД» и логируйте полное сообщение. В коде не храните пароль в открытом виде (комментарий: «использовать .env»).

## Вариант 16
Выполните запрос без подготовленного запроса (только для статических запросов): $pdo->query("SELECT COUNT(*) as cnt FROM users"). fetch()['cnt']. Выведите количество. Предупреждение в комментарии: для пользовательских данных всегда prepare.

## Вариант 17
bindParam с переменной: $stmt->bindParam(':value', $value, PDO::PARAM_STR). Измените $value перед execute(). Выведите результат — должна подставиться новое значение. Продемонстрируйте разницу с bindValue.

## Вариант 18
Обработка пустого результата: SELECT по несуществующему id. $row = $stmt->fetch(). Если $row === false — выведите «Запись не найдена». Иначе выведите данные.

## Вариант 19
Создайте таблицу через PDO (если разрешено): CREATE TABLE IF NOT EXISTS test (id INT AUTO_INCREMENT PRIMARY KEY, name VARCHAR(100)). execute(). Выведите «Таблица создана или уже существует».

## Вариант 20
Подключение с опцией PDO::ATTR_EMULATE_PREPARES => false (настоящие prepared statements на стороне MySQL). Выведите «Подключение с native prepared statements». Выполните простой SELECT и выведите одну строку.
